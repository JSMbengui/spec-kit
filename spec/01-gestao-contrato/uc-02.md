UC-02: Criar Ficha de Contrato
Utilizadores: Procurement
Prioridade: Crítica (Must Have)
Objetivo:
Permitir ao utilizador Procurement o registo completo da ficha de um novo contrato, vinculando-o a uma entidade já registada, definindo o valor adjudicado, o período de vigência e a fonte de financiamento (Banco Mundial ou AFD), de forma a garantir o controlo financeiro e a conformidade com os requisitos dos financiadores.

Fluxo Principal
1. O utilizador Procurement acede ao módulo “Contratos”.
2. Clica no botão “+ Novo Contrato”.
3. O sistema apresenta o formulário de criação de contrato.
4. O utilizador seleciona/obliga a vinculação a uma Entidade Contratante já registada no sistema (pesquisa por Nome ou NIF).
5. Preenche os dados principais do contrato:
  5.1 Número do Contrato (manual ou gerado automaticamente pelo sistema)
  5.2 Título/Descrição do Contrato
  5.3 Valor Adjudicado (em AOA)
  5.4 Data de Início do Contrato
  5.5 Data de Término do Contrato
  5.6 Fonte de Financiamento (seleção obrigatória: Banco Mundial ou AFD)
  5.7 Categoria do Contrato (Obras, Bens, Serviços, Consultoria)
  5.8 Outros campos relevantes (ex: objeto do contrato, responsável técnico, etc.)
6. O sistema realiza validações em tempo real.
7. O utilizador clica em “Guardar” ou “Criar Contrato”.
8. O sistema regista o contrato com estado inicial “Ativo”.
9. O utilizador é redirecionado para a ficha detalhada do contrato recém-criado.

Critérios de Aceitação
Item,Detalhe,Obrigatório
Vinculação a Entidade,Deve obrigatoriamente estar vinculado a uma Entidade Contratante já registada (UC-01),Sim
Data de Término,Deve ser posterior à Data de Início,Sim
Valor Adjudicado,Deve ser superior a 0,Sim
Fonte de Financiamento,Seleção obrigatória (Banco Mundial ou AFD),Sim
Categoria do Contrato,"Obras, Bens, Serviços ou Consultoria",Sim
Número do Contrato,Único no sistema,Sim
Estado inicial,Contrato criado como “Ativo”,Sim

Regras de Validação Adicionais
1. A Data de Término não pode ser anterior ou igual à Data de Início.
2. O Valor Adjudicado deve ser um valor positivo maior que zero.
3. O Número do Contrato deve ser único em todo o sistema.
4. A Fonte de Financiamento é de preenchimento obrigatório.
5. O sistema deve impedir a criação do contrato caso alguma validação crítica falhe.

Requisitos Não Funcionais
1. Auditoria: Todo o registo do contrato deve ser registado na trilha de auditoria (UC-A05), incluindo quem criou o contrato, data/hora, IP e valores inseridos.
2. Cálculo Automático: Após a criação, o sistema deve calcular e armazenar automaticamente o Saldo Disponível Inicial = Valor Adjudicado.
3. Rastreabilidade: Qualquer alteração posterior na ficha do contrato (ex: adendas) deve preservar o histórico.
4. Interface: Formulário responsivo com validação em tempo real (frontend + backend).
5. Permissões: Apenas o perfil Procurement (ou superior) pode criar novos contratos.

Campos Recomendados na Ficha do Contrato
Campos Obrigatórios:
  1. Entidade Contratante (pesquisa/select)
  2. Número do Contrato
  3. Título do Contrato
  4. Valor Adjudicado (AOA)
  5. Data de Início
  6. Data de Término
  7. Fonte de Financiamento (BM / AFD)
  8. Categoria (Obras / Bens / Serviços / Consultoria)

Campos Recomendados:
  1. Objeto / Descrição detalhada
  2. Responsável Técnico (Especialista)
  3. Data de Assinatura
  4. Estado atual do contrato
  5. Observações
