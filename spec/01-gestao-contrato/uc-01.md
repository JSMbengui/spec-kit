UC-A01: Autenticação com 2FA
Utilizadores: Administrador (Admin)
Objetivo:
Garantir um nível elevado de segurança no acesso à plataforma através de autenticação em dois fatores (2FA), tornando-o obrigatório para todos os utilizadores, conforme exigência do Banco Mundial.

Fluxo Principal

O utilizador acede à página de login e insere o email e a password.
O sistema valida as credenciais (email + password).
Se as credenciais estiverem corretas, o sistema verifica se o 2FA está ativado (obrigatório para todos).
O sistema solicita a introdução do código OTP (One-Time Password).
O utilizador insere o código de 6 dígitos gerado pela aplicação autenticadora (Google Authenticator, Microsoft Authenticator, Authy, etc.) ou recebido via SMS.
O sistema valida o código OTP.
Se o código estiver correto, o sistema emite um Token JWT com duração de 8 horas.
O utilizador é autenticado e redirecionado para o dashboard da aplicação.
O sistema regista o login completo na trilha de auditoria.


Critérios de Aceitação
Obrigatoriedade do 2FA

A autenticação com 2FA é obrigatória para todos os perfis de Administradores.
Não é possível desativar o 2FA.
No primeiro login após a ativação do 2FA, o utilizador deve configurar o autenticador (onboarding do 2FA).

Métodos de 2FA Suportados

Aplicação Autenticadora (TOTP) – Preferencial (Google Authenticator, Microsoft Authenticator, etc.)
SMS – Método alternativo (para utilizadores que não consigam usar app autenticadora)

Gestão de Tentativas Falhadas

Máximo de 5 tentativas falhadas do código OTP.
Após 5 tentativas incorretas, a conta deve ser bloqueada temporariamente por 30 minutos.
O bloqueio temporário aplica-se apenas ao 2FA (o utilizador ainda pode tentar login com email/password após os 30 minutos).

Token JWT

Expiração do Access Token: 8 horas (480 minutos)
O token deve ser renovável (refresh token mechanism recomendado)
O JWT deve conter claims mínimas: user_id, role, session_id, iat, exp
Implementar invalidação ativa de tokens (token revocation list) quando necessário

Auditoria

Registo completo de cada tentativa de login na trilha de auditoria, incluindo:
Sucesso ou falha
Tipo de falha (password incorreta ou OTP incorreto)
Endereço IP
User-Agent / Dispositivo
Timestamp
Método 2FA utilizado (TOTP ou SMS)



Fluxos Alternativos / Exceções

Password incorreta:
Mostrar mensagem genérica “Credenciais inválidas” e incrementar contador de tentativas de login.
Código OTP incorreto:
Mostrar mensagem “Código inválido ou expirado” e incrementar contador de tentativas de 2FA.
Conta bloqueada por 2FA:
Mostrar mensagem: “Muitas tentativas incorretas. A sua conta foi bloqueada temporariamente por 30 minutos.”
Utilizador ainda não configurou o 2FA:
Redirecionar para página de configuração inicial do 2FA (QR Code + código de backup).
Perda do dispositivo autenticador:
Prever fluxo de recuperação de 2FA (ex: códigos de backup ou contacto com Admin).


Requisitos Não Funcionais e Boas Práticas
Segurança:

Utilizar TOTP (Time-based One-Time Password) com algoritmo HMAC-SHA1 ou superior.
Segredo do 2FA deve ser armazenado de forma segura (encriptado).
Implementar rate limiting forte tanto no login como na validação do OTP.
Gerar códigos de recuperação (backup codes) de 8–10 dígitos durante a configuração do 2FA.
Proteger contra ataques de replay no OTP.

Experiência do Utilizador:

Interface clara com contador de tentativas restantes.
Opção de “Enviar código por SMS” como fallback.
Suporte a dark mode e responsividade (importante para utilizadores em campo).

Conformidade:

Atender às exigências de autenticação reforçada do Banco Mundial.
Recomenda-se alinhamento com normas como NIST 800-63B ou ISO 27001.
