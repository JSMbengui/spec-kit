UC-18: Gerir Alertas de Urgência
Utilizadores: Coordenador (principal), Admin, Procurement
Prioridade: Alta (Must Have)
Objetivo:
Fornecer ao Coordenador um widget centralizado e proativo com os alertas mais críticos do sistema, permitindo uma rápida identificação e resolução de problemas que possam comprometer a execução financeira e o cumprimento dos prazos contratuais com o Banco Mundial e AFD.


Fluxo Principal
1. O sistema agrega automaticamente todos os alertas ativos de diferentes fontes.
2. No Dashboard Estratégico, o widget “O que requer atenção?” ou “Alertas de Urgência” é carregado.
3. Os alertas são ordenados por criticidade (do mais urgente para o menos urgente).
4. O Coordenador visualiza os alertas agrupados em três categorias principais:
  4.1 Atrasos SLA (Estagnação)
  4.2 Vigência de Contratos
  4.3 Divergências / Faturas Rejeitadas
5. Ao clicar em cada alerta, o utilizador é redirecionado para a fatura ou contrato em causa.
6. Após resolver o problema (mover fatura, registar adenda, etc.), o sistema encerra automaticamente o alerta ou permite ao Coordenador marcá-lo como resolvido.
7. O sistema regista a resolução na trilha de auditoria.


Critérios de Aceitação
Categoria,Descrição,Critério
Atraso SLA,Faturas paradas mais de 48h num estado,Alerta gerado por UC-12
Vigência,Contratos com caducidade nos próximos 30 dias,Baseado em UC-04
Divergência,Faturas rejeitadas sem nova fatura substituta,Registo manual ou automático



Regras Gerais
1. Os alertas devem ser ordenados por nível de criticidade (Crítico → Alto → Médio).
2. Cada alerta deve mostrar:
  2.1 Título claro
  2.2 Descrição resumida
  2.3 Entidade / Contrato / Fatura envolvida
  2.4 Tempo decorrido ou dias restantes
  2.5 Responsável atual (quando aplicável)
3. O alerta deve desaparecer ou ser marcado como “Resolvido” quando a situação subjacente for corrigida.
4. Histórico completo de alertas (ativos e resolvidos) deve estar disponível.



Detalhe das 3 Categorias
1. Atrasos SLA (Estagnação)
  1.1 Faturas paradas > 48h (UC-12)
  1.2 Inclui escalamento após 72h
  1.3 Destaque em vermelho
2. Alertas de Vigência
  2.1 Contratos com menos de 30 dias para caducidade
  2.2 Contratos com saldo muito baixo (< 10%)
  2.3 Destaque em laranja/vermelho
3. Divergências
  3.1 Faturas rejeitadas (por técnico ou Procurement) sem que o empreiteiro tenha submetido uma fatura de substituição
  3.2 Faturas com parecer Desfavorável pendente de correção


Requisitos Não Funcionais
1. Notificações:
  1.1 Notificação push no sistema para o Coordenador quando surgirem novos alertas críticos.
  1.2 Email diário resumido (opcional, configurável).
2. Interatividade:
  2.1 Clique no alerta → navegação direta para a fatura ou contrato.
  2.2 Botão “Marcar como Resolvido” (com justificativa opcional).
3. Histórico:
  3.1 Secção separada “Histórico de Alertas” com filtro por data e categoria.
4. Atualização: Widget deve atualizar automaticamente (a cada 5-10 minutos ou em tempo real).



Integrações Fortes
1. UC-12: Alerta de Estagnação (48h)
2. UC-04: Painel de Vigência (SLA)
3. UC-13: Notificar Empreiteiro por Rejeição
4. UC-16: Visualizar KPIs Globais
5. UC-17: Funil de Faturas
6. UC-11: Trilha de Auditoria


Nota importante:
Este widget é um dos componentes mais visíveis do Módulo 5 – Dashboard de Gestão Estratégica. Ele transforma o sistema num instrumento de gestão proativa, em vez de apenas reativa, atendendo às boas práticas exigidas pelo Banco Mundial.
